Recording Controls and Model Selection Plan

Date: 2026-07-09

Goals

  1. Keep result actions and settings actions usable while the keyboard is open.
  2. Allow an active recording to be discarded before audio is processed.
  3. Make recommended and custom OpenRouter model selection practical on a phone.

Implementation

  1. Use a keyboard-avoiding app frame and keyboard-aware scroll views. Preserve action taps while an input is focused.
  2. Add an explicit Cancel recording action to the active recording panel. It will invalidate any pending operation, invoke the existing recorder cancellation API, and leave prior saved results intact.
  3. Keep Copy, Share, and Tags reachable while the editable result field has focus by resizing the app above the keyboard and allowing action taps without a preliminary keyboard-dismiss tap.
  4. Split model configuration into separate transcription and translation/cleanup sections. Persist a selected transcription model, send it to OpenRouter's transcription endpoint for voice flows, and retain the existing translation/cleanup presets and arbitrary custom model field.
  5. Offer compatible transcription recommendations for fast, balanced, and highest-quality use, while allowing an arbitrary OpenRouter transcription model ID. Let custom fields save through the keyboard Done action.

Verification

  1. Add screen tests for cancellation, keyboard scroll interaction props, and custom-model submission from the keyboard.
  2. Run focused tests, the full test suite, TypeScript checks, lint, and Expo diagnostics.
