Tarek Wisper Android V1 MVP Implementation Plan

Created July 6, 2026 from branch codex/android-v1-mvp, based on origin/feature/tarek-wisper-v1.

Goal: make the current Expo V1 MVP explicitly Android-ready without diverging from the iOS implementation or running an external Android build.

Approach: keep one shared React Native code path, add Android app metadata and permission configuration, close Android layout gaps, and extend tests around the Android-sensitive behavior. External EAS Android preview builds are intentionally out of scope until the user approves them.

Current Findings

  • app.json has iOS bundle metadata but no explicit android.package, android.versionCode, adaptive icon, or Android permission rationale.
  • npx expo config --type public currently resolves Android support and audio permissions through expo-audio, including android.permission.RECORD_AUDIO.
  • expo-audio is already used through an adapter in src/audio/audioRecorder.ts; the adapter can be tested without native private APIs.
  • Secure token storage uses expo-secure-store, SQLite uses expo-sqlite, and copy/share use Expo Clipboard plus React Native Share. These are cross-platform dependencies already covered by unit tests at the abstraction level.
  • The app wraps only the top safe area. Android gesture/navigation bars make bottom inset coverage around the tab bar worth fixing and testing.
  • Baseline checks pass after installing dependencies: npm test, npm run typecheck, npm run lint, and npm run doctor.
  • Local Node is 20.19.1; React Native 0.86 packages warn that they require 20.19.4 or newer.

Implementation Tasks

  1. Android Expo config. Update app.json with an Android package name, initial version code, adaptive icon using the existing icon asset, and explicit expo-audio plugin options for microphone permission and Android recording.
  2. EAS profile clarity. Add Android preview profile settings in eas.json that mirror the internal preview intent without invoking an Android EAS build.
  3. Safe-area/layout hardening. Update src/App.tsx so the app accounts for bottom safe-area insets around the tab bar while keeping existing top inset behavior.
  4. Tests. Extend Jest coverage for Android config shape, audio adapter permission options where practical, and bottom safe-area behavior. Keep tests deterministic and avoid native Android build dependencies.
  5. Verification. Run npm test, npm run typecheck, npm run lint, npm run doctor, and npx expo config --type public. Stop before any Android EAS preview build or deploy step.

Known Non-Goals For This Pass

  • No eas build --platform android, Play Store upload, or other external build/deploy action without user approval.
  • No native android/ directory generation unless later verification shows managed config is insufficient.
  • No broad UI redesign; Android changes should preserve the current V1 MVP experience.
