# Security Policy

## Supported versions

Security updates are provided for the latest release and the current `main` branch. Older releases may not receive fixes.

## Reporting a vulnerability

Do not open a public issue for a suspected vulnerability and do not include API keys, recordings, transcripts, email addresses, or other sensitive data in a report.

Use the private contact form at [saycopy.app/contact](https://saycopy.app/contact) and select **Privacy** or **Other**. Include a concise description and a safe way to reach you. If GitHub private vulnerability reporting is enabled for this repository, that channel is also appropriate.

Please include, when available:

- the affected version or commit;
- the relevant source paths;
- reproduction steps that use synthetic data;
- the expected security impact;
- any suggested remediation.

You should receive an acknowledgment within three business days. Please allow time for validation and a coordinated fix before public disclosure.

## Research expectations

Good-faith research must avoid accessing other people's data, degrading services, sending unsolicited messages, or testing against production with real credentials. Use local builds and accounts you control.
